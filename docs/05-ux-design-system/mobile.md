# Mobile

## Mobile priorities

- Critical alarm notifications must be actionable from mobile
- Fleet Overview and Device status are fully functional at mobile width
- Heavy data entry (device configuration, alarm rules) is desktop-first; mobile is read-optimised

## Touch targets

Minimum touch target is 44×44px. Alarm acknowledge and resolve actions use 56px targets.

## Breakpoints

| Name | Width | Target |
|---|---|---|
| Mobile | `< 640px` | Smartphones |
| Tablet | `640px – 1024px` | Tablets, HMIs |
| Desktop | `> 1024px` | Workstations |

## Mobile navigation

At mobile width, the navigation rail collapses to a bottom tab bar with 4 primary destinations: Overview, Alarms, Devices, Settings.
