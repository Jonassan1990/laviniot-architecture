# North Star

## The north star outcome

> **An operator at any facility connected to LavinIoT can see the real-time status of every monitored asset, receive an early warning before a failure occurs, understand why a threshold was breached, and act on an instruction — without needing a data scientist.**

This is the test of every product decision. If a proposed feature, interface, or module does not move toward this outcome, it does not belong in the current scope.

---

## How we measure progress toward the north star

| Dimension | Indicator |
|---|---|
| **Coverage** | Number of asset types the platform can connect to |
| **Signal quality** | Ratio of actionable alerts to total alerts |
| **Time to insight** | Time from event occurrence to operator notification |
| **Operator comprehension** | Can an operator act on a notification without analyst support? |
| **Edge readiness** | Can the platform function during cloud outages? |

---

## What comes after the north star

The north star defines the destination. The [Roadmap](/product/roadmap) defines the path.

Phase 1 establishes the foundation: connectivity, telemetry storage, and basic alarming.
Phase 2 adds intelligence: AI-assisted anomaly detection and predictive signals.
Phase 3 adds scale: multi-site, multi-tenant, compliance-ready deployments.

Each phase is evaluated against the north star. A phase is complete when it measurably advances the outcome, not when a checklist of features is shipped.
