# Release Notes

> Release notes are the customer-facing and team-facing record of what shipped. They are linked to the Sprint Journal entries and ADRs that caused each change.

---

## Purpose of this section

Release notes capture what changed in each release — for the platform, the dashboard, the API, and the marketing site. They are traceable: every significant change links to a Sprint Journal entry or ADR.

## Release note format

Each release note includes:
- **Version / identifier**
- **Component**
- **Release date**
- **What changed** (by category: new, changed, fixed, removed)
- **Links to relevant ADRs and Sprint Journal entries**
- **Known issues** (if any)
