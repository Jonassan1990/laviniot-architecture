# Design Principles

## 1. Operators first

Every interface is evaluated from the operator's perspective. If an operator cannot act on it without analytical training, it needs to be redesigned.

## 2. Clarity over density

Industrial environments involve high stakes and frequent interruptions. Interface clarity takes precedence over information density. One clear action is better than ten available options.

## 3. Alarm states are never ambiguous

Alarm severity is conveyed by colour AND symbol AND text. A red screen is not enough. A colour-blind operator must receive the same urgency signal as any other.

## 4. The interface earns trust incrementally

The platform does not overwhelm operators with automated recommendations from day one. Trust is earned through demonstrated accuracy. AI outputs are labeled, explained, and optional.

## 5. Industrial environments are not offices

Interfaces are used on large screens at distance, on tablets in the field, and on mobile in loud, bright environments. Touch targets are large. Contrast is high. Font sizes are generous.

## 6. Actions are explicit, not implied

Destructive or consequential actions (acknowledge alarm, resolve alarm, deregister device) require explicit confirmation. No swipe-to-delete patterns in an industrial platform.
